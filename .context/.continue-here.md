### Correcciones durante implementación
- Tabla `productos` NO tiene columna `descripcion` ni `propositoprincipal` directo — esos datos están en `productoscaracteristicas`
- Tabla `productosxcatalogo` NO es la relación correcta para filtrar productos por cliente — la tabla `productos` tiene `clienteid` y `zonaid` directamente
- Material de etiquetado se divide en: **empaque** (`tipomaterialid=1`) y **envase** (`tipomaterialid=2`)
- La tabla `catalogos` NO es catálogo de productos — es para opciones de procesos (dropdownlist). Los catálogos de productos se forman por Cliente+Zona

<decisions_made>

### API
- API key se acepta por header (`x-api-key`) O por query param (`apiKey`) para facilitar pruebas en navegador
- Cliente Supabase separado para API (`lib/supabase-api.ts`) con service role key, sin cookies
- Middleware excluye `/api/v1` del auth de sesión — la API tiene su propia autenticación
- Productos se filtran por `clienteid` y `zonaid` directamente en tabla `productos` (no via `productosxcatalogo`)
- Materiales empaque/envase son la misma tabla `materialesetiquetado` filtrada por `tipomaterialid` (1=empaque, 2=envase)
- Vistas accesibles via endpoint dinámico con whitelist de seguridad

### Cotizaciones (decisiones anteriores)
- Costo MP viene de `productos.mp`
- Total Costo viene de `productos.costo` (MP + ME + MEM + MS)
- Costo MS viene de `productos.ms`

</decisions_made>
